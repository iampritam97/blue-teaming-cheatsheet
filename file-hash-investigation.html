Used to identify and classify malicious files using MD5, SHA1, or SHA256 hashes during malware, phishing, and endpoint investigations.

When to Investigate a File Hash

Detected by EDR / Antivirus on an endpoint
Extracted from phishing email attachments
Observed in process execution or persistence
Shared via threat intelligence or SOC alert

Hash Reputation & Classification

These platforms help determine whether a file hash is known malware, suspicious, or previously observed in threat campaigns. MITRE: T1204

VirusTotal MalwareBazaar Hybrid Analysis ThreatFox AlienVault OTX

Malware Context & Attribution

These tools provide deeper context such as malware family, campaign associations, threat actor attribution, and behavioral insights.

ThreatMiner InQuest Labs Intezer Triage

SOC Analyst Notes

Always validate hashes using multiple platforms, not a single source.
If a hash is unknown, proceed with dynamic sandbox analysis.
Correlate hash detections with process execution, network traffic, and persistence.
Document findings and update SIEM / EDR detection logic.

Used to identify and classify malicious files using MD5, SHA1, or SHA256 hashes during malware, phishing, and endpoint investigations.

What Is a File Hash?

A file hash is a fixed-length cryptographic fingerprint generated from a file‚Äôs contents.
Even a 1-byte change in the file results in a completely different hash value.
Hashes are used to uniquely identify files, verify integrity, and detect known malware.

Common Hash Algorithms Used in SOC

MD5
Fast but cryptographically weak.
Still widely used for malware lookups.
SHA-1
Stronger than MD5 but deprecated for security.
Common in legacy tools.
SHA-256
Industry standard for malware identification.
Preferred for threat intelligence sharing.
SHA-512
Rare in SOC workflows.
Mostly used in research contexts.

Hash Extraction Commands (Windows & Linux)

These commands are commonly used by SOC analysts to calculate file hashes directly from endpoints, forensic images, or malware samples.

Windows (Built-in)

certutil -hashfile sample.exe MD5
certutil -hashfile sample.exe SHA1
certutil -hashfile sample.exe SHA256

Available by default on modern Windows systems.

Windows (PowerShell)

Get-FileHash sample.exe
Get-FileHash sample.exe -Algorithm SHA256

Preferred in PowerShell-based investigations.

Linux / macOS

md5sum sample.bin
sha1sum sample.bin
sha256sum sample.bin

Standard utilities available on most Unix-based systems.

Cross-Platform (OpenSSL)

openssl md5 sample.bin
openssl sha1 sample.bin
openssl sha256 sample.bin

Useful in restricted or minimal environments.

Important Limitations of Hash-Based Detection

Hashes fail against polymorphic or packed malware.
Attackers can easily change hashes by recompiling binaries.
Unknown hashes do not mean benign.
Hashes must be correlated with behavior, network traffic, and execution context.

When to Investigate a File Hash

Detected by EDR / Antivirus on an endpoint
Extracted from phishing email attachments
Observed in process execution or persistence
Shared via threat intelligence or SOC alert

SOC Analyst Notes

Always validate hashes using multiple platforms, not a single source.
If a hash is unknown, proceed with dynamic sandbox analysis.
Correlate hash detections with process execution, network traffic, and persistence.
Document findings and update SIEM / EDR detection logic.

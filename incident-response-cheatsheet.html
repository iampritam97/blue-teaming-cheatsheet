üö® Incident Response Quick Cheat Sheet

A fast, practical reference for SOC analysts to follow during active security incidents.

Initial Triage

First 5‚Äì10 minutes of an alert. Goal: decide is this real? and how bad is it?

Identify IOC type: IP, URL, file hash, email, or user account
Validate alert severity and confidence level
Check if activity is ongoing or historical
Look for signs of exploitation or user interaction

Investigation

Deep-dive analysis to understand scope, impact, and attacker behavior.

Correlate IPs, domains, URLs, hashes, and email artifacts
Identify affected users, hosts, and network segments
Review authentication logs for suspicious access
Map activity to MITRE ATT&CK techniques
Look for lateral movement or privilege escalation
Determine attack timeline and entry point

Containment

Stop the attack from spreading while preserving evidence.

Block malicious IPs, domains, URLs, and file hashes
Isolate compromised endpoints using EDR
Disable or reset compromised user accounts
Apply temporary firewall or proxy rules if needed

Eradication & Recovery

Remove attacker presence and restore business operations safely.

Remove persistence mechanisms (services, registry, scheduled tasks)
Patch exploited vulnerabilities or misconfigurations
Reimage or restore systems from clean backups if required
Monitor systems closely for reinfection

Post-Incident Activities

Improve security posture and prevent similar incidents.

Document incident timeline, root cause, and impact
Update SIEM detection rules and alert thresholds
Add new IOCs to threat intelligence platforms
Conduct lessons-learned and analyst training sessions
